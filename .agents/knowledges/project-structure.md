### Project Tech Stack
- **Language**: Go (Golang) 1.21+
- **Primary Framework**: Custom wrapper via `ego` package
- **Code Architecture**: Clean Architecture with Usecase and Repository patterns
- **Database**: MySQL 8.0+
- **Testing Framework**: Go standard testing package with mockery for mock generation
- **Version Control**: Git
- **Containerization**: Docker
- **Shared Library**: Use the `ego` library for cross-cutting concerns (DB connections, Logging, Tracing, REST calls, Error codes, Validator, etc). Do not write custom wrappers for these.
- **Inter-service Communication**: REST

#### cmd
`cmd` directory is the entry point of the application. It is the place where the application is initialized and the server is started.

#### docs
`docs` directory is the documentation of the application. It is the place where the documentation of the application is stored. generated by `swagger`

#### config
`config` directory is the configuration of the application. It is the place where the configuration of the application is stored.

#### app
`app` directory is the main directory of the application. It is the place where the application is initialized and the server is started.
